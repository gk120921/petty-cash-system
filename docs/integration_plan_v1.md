# AI 優化實驗室：整合管理系統 (Nexus) 規劃書

## 1. 專案願景
本整合系統（代號：Nexus）旨在將「零用金」、「印度倉庫」與「採購審查」三個獨立模組整合為一個統一的營運平台。透過矩陣式 Agent 協作架構，建立一個具備高美學、高穩定性且數據互通的中央指揮塔。

## 2. 核心團隊架構 (The Matrix Structure)
我們將團隊分為四個獨立但互通的 **SQUAD (小隊)**，每個小隊均具備完整的五人配置：

| 角色 | 職責說明 (Agent Role) | 核心目標 |
| :--- | :--- | :--- |
| **👑 規劃者 (Planner)** | 需求邊界定義、排程控管、商業邏輯檢核。 | 確保開發方向不偏離商業價值。 |
| **🎨 前端工程師 (Frontend)** | React/Vite 架構實作、Premium 介面呈現、響應式設計。 | 提供極致的視覺美學與流暢操作。 |
| **⚙️ 邏輯工程師 (Logic)** | 核心演算法、複雜資料處理流程、前端狀態管理。 | 確保系統「大腦」運作精確無誤。 |
| **🏗️ 後端工程師 (Backend)** | API 設計、資料庫穩定性、安全性驗證、效能優化。 | 建構強韌的數據基礎設施。 |
| **🧪 QC 工程師 (QC)** | 邊界案例測試、自動化掃描、穩定性驗證。 | 達成零崩潰 (Zero Crash) 的產品品質。 |

## 3. 系統分組與分工 (System Allocation)

### 🔴 小隊 A：零用金系統 (Petty Cash Squad)
- **重點：** 精確的財務對帳邏輯、OCR 辨識流程優化、多角色權限控制 (RBAC)。
- **挑戰：** 確保每一筆帳務異動的不可逆性與記錄完整性。

### 🔵 小隊 B：印度倉庫系統 (India WMS Squad)
- **重點：** 雙語 (中/英) 自動切換、六位小數的高精度 BOM 換算、批次追蹤 (MRB)。
- **挑戰：** 處理複雜的生產入庫與領料邏輯。

### 🟢 小隊 C：採購審查系統 (Procurement Squad)
- **重點：** 多級審批工作流、組織架構連動、歷史紀錄歸檔。
- **挑戰：** 確保動態簽核流程的靈活性與透明度。

### 🌟 小隊 D：整合系統小隊 (Integration Master Squad)
這是最關鍵的團隊，負責將以上三個「孤島系統」串聯起來：
- **規劃者：** 設計「全域儀表板 (Global Dashboard)」，定義跨系統的數據標準。
- **前端工程師：** 開發「統一入口網關 (Portal)」，讓使用者在單一介面流暢切換各系統。
- **邏輯工程師：** 處理跨系統的數據同步邏輯（例如：採購通過後自動同步至倉庫預期入庫）。
- **後端工程師：** 建立「統一身份認證 (SSO)」，實現一次登入、全域通行。
- **QC 工程師：** 進行「整合集成測試 (SIT)」，確保跨系統呼叫不會產生衝突。

## 4. 協作通訊協定 (Communication Protocol)
為了確保這 20 個 Agent 能高效運作，我們將遵循以下規則：
- **數據統一化：** 所有系統的 API 格式、錯誤碼定義須由「整合小隊」統一定義標準。
- **模組化原則：** 遵循 **[500行規則]**，確保程式碼高度解耦，方便整合系統調用。
- **同步機制：** 採用事件驅動架構 (Event-Driven)，當 A 系統發生變更，透過整合小隊的「邏輯匯流排」通知 B 或 C 系統。
