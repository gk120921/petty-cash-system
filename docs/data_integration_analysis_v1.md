# 印度端子工廠：跨系統數據整合分析報告 (V1.1)

## 1. 分析背景
本報告基於 Agent 團隊對「零用金」、「採購審查」及「印度倉庫」三套系統資料庫 (SQLite/JSON) 的深度掃描。本系統定位為 **「印度端子工廠 (India Terminal Factory) 專用管理系統」**，所有邏輯將以此為核心出發點。

## 2. 核心整合點分析

### 2.1 供應商管理：分層制度 (Supplier Tiering)
- **零用金系統:** 
    - **特點:** 追求快速輸入、流程簡便。
    - **邏輯:** 不需要強制關聯「合格供應商」，支援手動快速輸入，以應對工廠現場的瑣碎支出。
- **採購管理系統:**
    - **特點:** 嚴謹審核、品質控管。
    - **邏輯:** 必須為 **「合格供應商」**，具備完整的統編、資信與聯絡資訊。
- **整合建議:** 整合系統提供「供應商主檔」，但允許零用金模組進行「非清單內採購」。

### 2.2 會計科目與費用歸類
- **定位:** 服務於 **印度端子工廠** 的財務成本分析。
- **整合建議:** 
    - 以採購系統的 `accounting_subjects` 為主檔。
    - 產出工廠整體的「製造費用與運營成本分析報表」。

### 2.3 部門主管與審核差異化
- **零用金審核邏輯:** 
    - **金額門檻:** 固定金額以下不需開單審查，亦無階層審查限制。
    - **簡化原則:** 不需強制輸入統編，以提升工廠基層運作效率。
- **採購審核邏輯:** 
    - 維持嚴謹的階層審核與 PR/PO 連動機制。
- **整合建議:** 由整合系統統一管理部門樹，但針對不同模組套用不同的「工作流模板 (Workflow Template)」。

### 2.4 採購轉入庫：工廠生產聯動 🌟
- **邏輯:** 當印度工廠採購的端子原料單 (PO) 核准後，自動生成 WMS 的「預期入庫」紀錄。
- **預期效益:** 確保工廠生產排程能精確預知原料到貨時間。
